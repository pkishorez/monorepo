`std-toolkit` is one package with a subpath per component. Each component is a piece of the story from [Why](/docs/std-toolkit): entities live in single-table item collections, evolve their shape safely over time, and sync into the browser. This page walks through those components one by one.

## core

Common utilities shared by every other component — most importantly, the **Entity** vocabulary. An entity is `{ value, meta }`, where meta carries the entity type (`_e`), schema version (`_v`), recency (`_u`), and tombstone (`_d`) fields. core defines this envelope once (`EntitySchema`, `MetaSchema`), plus `Broadcaster` and the base `StdToolkitError`, so every adapter and the sync engine speak the same language instead of redefining it.

→ [core](/docs/std-toolkit/core)

## eschema — evolving schema

Persisted data outlives its schema. eschema makes the schema itself carry its history: a chain of versions `v1 … latest`, where old data is folded forward to the current shape on read — no migration scripts, no downtime. core's `_v` meta field is an eschema version.

→ [eschema — Why](/docs/std-toolkit/eschema)

## tanstack-sync

The abstraction for TanStack DB integration: `createStdSync` mirrors server item collections into live browser collections, with pluggable sync strategies, paced optimistic writes, and offline storage (IndexedDB) so state survives a reload.

→ [tanstack-sync](/docs/std-toolkit/tanstack-sync)

## Database adapters

Two implementations of one storage kernel — the same partition-key / sort-key / item-collection topology, so swapping the database never renegotiates meaning:

- **DynamoDB** — the pattern's birthplace and the production target: table/entity services, type-safe expression builders, marshalling utilities. → [dynamodb](/docs/std-toolkit/dynamodb)
- **SQLite** — the same single-table services over SQLite, with adapters for Node, Bun, better-sqlite3, and Cloudflare Durable Objects. Tests, local dev, and edge runtimes without mocks. → [sqlite](/docs/std-toolkit/sqlite)

## Install

```sh
npm install std-toolkit
npm install effect                     # required by all subpaths
npm install @tanstack/react-db react   # required by tanstack-sync
```

Requires Node ≥ 24.

## Bin

```sh
npx eschema          # schema evolution CLI (see std-toolkit/eschema)
```
