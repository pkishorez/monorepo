`std-toolkit/tanstack-sync` provides TanStack DB sync helpers for Effect-based server APIs. Keyed collections use `sync`; singleton collections use `singleItemSync`. An optional Offline Storage adapter persists the engine-owned Source of Truth and Sync State, backing convergence and tombstone retention rather than acting as a hydration-only cache.

## Install

```sh
npm install std-toolkit effect @tanstack/react-db react
```

```typescript
import {
  createStdSync,
  syncStrategy,
  singleItemSyncStrategy,
  paceStrategy,
} from 'std-toolkit/tanstack-sync';
import { idbStorage } from 'std-toolkit/tanstack-sync/offline-storage/idb';

const std = createStdSync({
  offlineStorage: idbStorage({ name: 'app-sync', version: 1 }),
});
```

## Keyed sync

Mirror a whole entity set with a strategy, and optionally load partitions on demand:

```typescript
const tasksCollection = createCollection(
  std.sync({
    schema: TaskSchema,
    strategy: syncStrategy.oldToNew({
      fetch: ({ cursor }) => api.getTasks({ cursor }),
    }),
    updatePacing: paceStrategy.coalesce({ wait: 50 }),
    onInsert: (task) => api.createTask(task),
    onUpdate: ({ id, updates }) => api.updateTask(id, updates),
    onDelete: (id) => api.deleteTask(id),
  }),
);
```

## Single item sync

Use `singleItemSync` for exactly one record, such as app settings or a profile.

```typescript
const settingsCollection = createCollection(
  std.singleItemSync({
    schema: SettingsSchema,
    strategy: singleItemSyncStrategy.getOnce({ get: () => api.getSettings() }),
    onUpdate: ({ updates }) => api.saveSettings(updates),
  }),
);
```

## Offline storage and the registry

Root storage is inherited by collections that omit `offlineStorage`; a collection can opt out with `offlineStorage: false` for collection-local memory. The `std.registry()` routes broadcast envelopes to collections created by the same `createStdSync` instance — `persist: true` writes server truth through Source of Truth, `persist: false` only projects to mounted collections.
