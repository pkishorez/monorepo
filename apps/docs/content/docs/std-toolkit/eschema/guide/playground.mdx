One evolving schema, three versions:

```ts
const User = ESchema.make({ name: Schema.String })
  .evolve('v2', { email: Schema.NullOr(Schema.String) }, (prev) => ({
    ...prev,
    email: null,
  }))
  .evolve('v3', { name: null, fullName: Schema.String }, (prev) => ({
    fullName: prev.name,
    email: prev.email,
  }))
  .build();
```

Pick a stored row below and watch how decode folds it forward to the latest shape — including a legacy row with no `_v` stamp at all.

<ESchemaPlayground />

Every path ends in the same place: application code only ever sees the v3 shape, no matter when the row was written. That is the whole trick — the schema carries its history, so reads absorb it.
